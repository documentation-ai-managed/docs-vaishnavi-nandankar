## Overview

Vaishnavi Nandankar provides a centralized documentation space where you organize, manage, and share project documentation effortlessly. You create structured pages, collaborate with teams, and maintain version control using familiar Markdown and MDX syntax. This platform scales with your projects, from simple READMEs to comprehensive API references.

<Callout kind="info">
  Start by exploring the sidebar navigation to find guides tailored to your needs.
</Callout>

## Quick Start

Get up and running in minutes. Follow these steps to create your first documentation page.

<Steps>
  <Step title="Sign Up" icon="user-plus">
    Visit `https://vaishnavi-nandankar.com` and create your account using email or GitHub.
  </Step>
  <Step title="Create Workspace" icon="plus">
    Set up a new documentation space for your project. Choose a name and add initial pages.
  </Step>
  <Step title="Edit Page" icon="edit-3">
    Open the editor and write in Markdown. Preview changes live.
  </Step>
  <Step title="Publish" icon="globe">
    Save and publish. Share the public link with your team.
  </Step>
</Steps>

<CodeGroup tabs="Markdown,MDX">
```markdown
# Welcome

This is your first page.
```

## Brand Guidelines

Maintain visual consistency across your documentation. Use the primary brand color `#3B82F6` for accents.

<Callout kind="tip" default-open="true">
  **Primary Color:** `#3B82F6`<br/>
  Apply it in CSS: `color: #3B82F6;` or `--primary: #3B82F6;`.
</Callout>

```css
:root {
  --brand-primary: #3B82F6;
  --brand-text: #1F2937;
}

.button {
  background-color: var(--brand-primary);
  color: white;
}
```
